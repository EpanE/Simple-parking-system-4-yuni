📋 main_system.ino — setup() and loop(), in execution order
#StepDetail
0setup() (once)SPI/RFID init, sensor + LED pin config, IR pin config, LCD init ("System Ready"), gate closed, optional WiFi/Blynk connect
1Blynk.run()Only if ENABLE_BLYNK — services the Blynk connection each iteration
2Poll sensorsreadDistance() on all 3 HC-SR04 → occupied if <10cm → set that slot's RED/GREEN LED → track nearest
3Update LCDRow 2 shows the nearest occupied slot's label + distance, or "No object"
4RFID entry checkIf a card is present: read UID → isWhitelisted()? → "Access OK" + openGate(), or "Access Denied" (no gate)
5IR exit checkIf digitalRead(IR_PIN) == LOW: openGate() — no authorization needed, same shared gate
6Blynk pushOnly if ENABLE_BLYNK and ≥1000ms since last push — sends free-slot count to virtual pin V0
7delay(200)Then loop() repeats from the top
—BLYNK_WRITE(V1)Async callback, outside loop() — a "Button" press in the Blynk app also calls openGate() directly
⚠ openGate() blocks the whole loop for 3000ms. It's one shared function called from three places (RFID grant, IR trigger, Blynk V1 button) — while it's running, sensor polling, LED updates, RFID scanning, IR checks, and the Blynk push all pause. The gate can only be triggered once at a time; a new trigger during those 3s has no effect until the current cycle finishes.
